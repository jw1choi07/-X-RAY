  - judge 프롬프트 자체는 유지(부작용은 없고, VALID/INVALID 판단 근거가 더 명확해졌으니 유지할 가치는 있음) — 다만 이걸로 매칭 정확도가 더 오를 것으로 발표에서 주장하지 않기로 함.
- **Groundedness 체크 fuzzy 매칭 적용**: 공백만 제거하던 `normalizeForMatch()`를 구두점/기호(따옴표, 대시, 가운뎃점 종류, 괄호 등)까지 제거하도록 강화 — 내용 문자(한글/영문/숫자)는 그대로 정확 일치를 요구해서 진짜 다른 내용이 통과하는 일은 없도록 함.
  - 21개 서비스 재측정: 원문근거확인 **62.3%**(직전 57.7~61.4% 범위에서 소폭 상승), 매칭 타당성 47.2%(비슷한 범위 유지).
  - **정직한 결론**: 방향은 맞지만 극적인 개선은 아님 — 실행별 변동폭(±5%p 정도) 안에 있어서 "확실히 좋아졌다"고 단정하긴 이름. 그래도 부작용 없고 이론적으로 타당한 개선이라 유지.
- **멜론 프리셋 문서 재크롤링 시도** → 자동화로는 실패: 실제 개인정보처리방침 콘텐츠가 **크로스 오리진 iframe** 안에 있어서, robots.txt 확인 후 재크롤링해도(Playwright 렌더링 포함) 여전히 350자짜리 nav 껍데기만 잡힘. 브라우저 자동화로 iframe URL을 알아내려 했으나 브라우저 보안 정책(cross-origin) + 확장 프로그램 안전 필터(쿠키/쿼리스트링 포함 URL 차단) 둘 다에 막힘.
  - **해결**: 사용자가 브라우저에서 직접 복사해서 전달 → `멜론_개인정보처리방침.txt`(24,872자, 정식 17개 조항 전문) 교체, 프리셋 임베딩 캐시도 재생성. 분석 테스트 결과 7건 정상 판정 확인 — 이제 UI 필터 통과해서 프리셋 목록에도 다시 노출됨(홈 화면 멜론 카드가 이제 진짜 문서로 분석됨).
